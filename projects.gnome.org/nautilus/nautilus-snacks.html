Snack-size Nautilus Hacks

This is a list of some Nautilus hacks that would be nice to have for 2.6. They aren't really bite-size - a list of bite-sized tasks would be too hard to maintain. These tasks will take a certain amount of research and planning, and will almost certainly need a few rounds of review to get right.

Selection-sensitive side pane

The side pane is currently geared toward showing information about the current directory. It would be nice for it to try to show more about the selected files. Some possibilities:

  • Show a "zoomed in" version of each file in the Information side pane. Show all of the information types from the Icon Captions for each selected file.
  • Maybe provide all of the Open As actions on the side pane for the selected files
  • Change the notes pane to show notes for selected files instead of just the current file/directory. This will make notes much more useful.

Things and files to look at:

  • Nautilus file attributes, and reading them asynchronously. See nautilus/docs/nautilus-io.txt
  • Registering for and receiving selection change notifications, nautilus/libnautilus/nautilus-view.h, nautilus/src/nautilus-window-manage-views.c (change_selection)
  • nautilus/src/nautilus-information-panel.c
  • nautilus/components/notes/

Rearrangable list-view columns

This would have two parts: First, all of the things that you can show as icon captions in the icon view should be made available as list view columns. Second, there should be a way to edit which columns are available. There is a column editor in libegg, this could probably be integrated. Or maybe the columns could use the same choices/priorities as the icon view.

Things and files to look at:

  • Nautilus file attributes, and reading them asynchronously. See nautilus/docs/nautilus-io.txt
  • libegg/libegg/column-chooser/
  • nautilus/src/file-manager/fm-list-view.c
  • nautilus/src/file-manager/fm-list-model.c

Recursive permissions

The way MacOSX does this makes sense - they have a button underneath the permissions that says "Apply to Folder Contents" or something like that. This screenshot shows a possible new layout for permissions. One benefit of this layout is that "Read And Write" is more easy to apply recursively, because a Read and Write file should probably be +rw, while a Read and Write directory should probably be +rwx.

Things and files to look at:

  • nautilus/src/file-manager/fm-properties-window.c

Glob Selection Dialog

People want to be able to perform operations on sets of files. Rather than implement filtering, we want to implement a simple dialog that lets users specify a pattern to search for. When they're done, the file window should select all files that match this pattern.

Things and files to look at:

  • nautilus/src/file-manager/fm-directory-view.c
