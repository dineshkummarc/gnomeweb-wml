Problems - What do to do if Chronojump or Chronopic don't work

Printable version
IMPORTANT:
If you experience any problem with Chronojump software, you need to check if your installed version is last Chronojump version. Maybe the problem you found is corrected on that version. We will not help to users who have'nt updated the software. Last version is: 0.8, released on: December-2-2008. Know your version.
  1. Chronojump wont start
  2. Chronopic don't work
  3. Know your Chonojump installed version
  4. Check Chronopic port in MyPC
  5. Force port to COM1 or COM2
Chronojump won't start
Different things can happen here, maybe installation haven't finished, maybe database was not created completely, maybe is there any problem with the paths, maybe Mono program was deleted, or maybe there's any security problem with your antivirus or operating system.
Please follow the path shown on Chronojump Won't start Flow Chart, and write down what it worked for you. Here there's help on the actions of the flow chart:

Know your Chronojump installed version
See Know your Chonojump installed version
Execute Chronojump-mini
On Windows Start Menu, inside Chronojump folder, you will have a entry for Chronojump-mini. Click on it.
Execute Chronojump-mini.bat
Navigate to Chronojump folder (maybe at C:\Program Files\Chronojump\chronojump-0.8\windows) and double click on chronojump-mini software.
Execute Mono Command Prompt
On Windows Start Menu, inside "Mono x.y for Windows" folder, you will have an entry caled: "Mono-x.y Command Prompt". Click on it.
On Mono Command Prompt, go to Chronojump folder\data
This is the most complicated operation.
If Chronojump is installed at C:\Program Files\Chronojump\chronojump-0.8, then do:
write "cd C:\" and press ENTER. You should see "C:\>"
write "cd Program Files\Chronojump\chronojump-0.8\data" and press ENTER. You should see "C:\Program Files\Chronojump\chronojump-0.8\data>"
mono chronojump.prg
Write "mono chronojump.prg" and press ENTER
mono chronojump_mini.prg
Write "mono chronojump_mini.prg" and press ENTER
Delete database
Do this if you have not valuable data in database (you have'nt started to use Chronojump seriously).
If Chronojump was installed where specified above, then delete file: C:\Program Files\Chronojump\database\chronojump.db
Write developer
Write an email to , and include your operating system, information about the flow chart result, the path (A, B, C, D, E, F) you followed on the chart, and the first and last (sorted by date) files in logs folder: C:\Program Files\Chronojump\database\logs. Thanks, your help is appreciated.

Chronopic don't work
Notice first that once Chronopic is connected, you need to create or open a session and create or add persons in order to start doing tests.

Please follow the path shown on Chronopic don't work solving Flow Chart, and write down what it worked for you. Here there's help on the actions of the flow chart:

Know your Chronojump installed version
See Know your Chonojump installed version
Disconnect attached hardware
Please, disconnect any contact platform or other hardware that's connected to the Chronopic.
Have only one connection: Chronopic to the computer using the USB cable.
Check lights
Check that light named D1 brights all the time except when Chronopic TEST button is pressed.
Write developer asking new Chronopic
Write an email to . We advertise you that all the Chronopics we have sold have been tested before.
Disable antivirus
We have documented that at least antivirus Spy-doctor interferes with Chronopic. We are sure that other's don't work properly. Please ensure no antivirus software is working while doing this tests.
Check port in MyPC
See Check Chronopic port in MyPC
Write new port in preferences
On Chronojump's menu, File/Preferences, write the port you found. Eg: COM7
Force port to COM1 or COM2
See Force port to COM1 or COM2
Check it's changed
Close all MyPC related windows. Disconnect Chronopic USB cable, wait two seconds and reconnect again. Check using again: Check Chronopic port in MyPC to see if detected port is now COM1 or COM2 as defined on "Force port to COM1 or COM2" above.
Changes in BIOS
Ask help to a computer related friend, on how to find if the BIOS of your computer comes with default deactivation of serial or USB-serial ports.

Up

Know your Chonojump installed version
If you want to know your Chronojump installed version, you should start Chronojump and clic on "Help" and "About" on the menu.
If for any reason you cannot start Chronojump, go to installed directory (maybe C:\Program Files\Chronojump) and see what directories are inside that folder. If you have a directory called "chronojump-0.7" then "0.7" is your installed version. If you have several directories like that it means you installed different versions of the program. Your version will be the highest number.

Up

Check Chronopic port in MyPC
Do the following:
  1. Click with the right button on "MyPC" icon at desktop or Start Menu
  2. Select "properties" (last option)
  3. Go to Hardware
  4. Select "administrate hardware" or "administrate dispositives" (or something similar in english, in spanish is "administrador de dispositivos"). It's first button.
  5. Click on the '+' at left of COM & LPT ports
  6. The portname will be what it's written like COM? on the USB-serial line. Eg: if it's written COM7, then you should write COM7 at Chronojump preferences.

Up

Force port to COM1 or COM2
Repeat the same as Check Chronopic port in MyPC, but:
  1. On the line where the port name is shown, right click and select "properties" (last option)
  2. Port configurations
  3. Advanced Options
  4. Select COM1 or COM2 on the list shown on that window.
If COM1 and COM2 are "used", then select unused ports below 10. If doesn't work, try to select the COM1 or COM2 (normally they are not really "used").
