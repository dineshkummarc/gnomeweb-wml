The New Stable Gnumeric 1.10.x Series is Now Available.

The Gnumeric Team is proud to announce the 1.10.x series. This is the product of almost two years of development.

What's New?

  • Adjustable sheet size: sheets are no longer restricted to 256 columns of 65536 rows.
  • Graphs have been improved:
    • New plot types for surfaces and probability plots.
    • When an axis is a date or time axis, reasonable places for tick marks will be picked.
  • ODF import has been improved and Gnumeric now supports almost all functions in the large function group. Note, though, that we name some of them differently and remap the names on import and export. Note, though, that until and unless the syntax and semantics of ODF files are defined, interoperability between any two implementations will be limited. Five years with no action on that front suggests that it will not happen in the near term.
  • Faster evaluation: some fairly common sheets with large numbers of calls to the HLOOKUP or similar functions, over the same database have been improved from "rather slow" to "instant". Similar improvements have been implemented for farms of RANK or PERCENTILE calls.
  • The memory usage for large sheets is significantly down.
  • The statistical analysis tools have been improved and there are quite a few new ones, for example:
    • Kaplan-Meier Estimates
    • Principal Component Analysis
    • Normality Tests
    • Sign Tests
    • Trend and/or Season Corrected Exponential Smoothing
  • Support for sheet objects (lines, arrows, widgets, ...) has been improved.

Requirements

Gnumeric 1.10.x requires the new Goffice 0.8.x library series, a fairly recent libgsf, and a GTK+ stack. See here for details.
