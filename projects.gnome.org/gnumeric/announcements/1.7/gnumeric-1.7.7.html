Gnumeric 1.7.7 aka "Quiet Before the Storm" is now available.

The Gnumeric team is proud to announce the release of Gnumeric 1.7.7, a development release fixing a number of smaller issues.

This release requires the concurrently released Goffice 0.3.6, see below.

  • Jody
    • Patch a leak in DataTables.
    • Fix AutoFormat loading.
    • Add support for the standard builtin formats in xlsx.
    • Fix col/row grouping deletion. [Bug 388505].
  • Morten
    • Unify layout and string value formatting.
    • Make sheet duplication handle sheet-local names.
    • Make sheet duplication change sheet references to point to the new sheet. [Bug 158825, Bug 158691].
    • Fix parsing of new workbook-level named expresions. [Bug 306686].
    • Fix xml loading of names that refer to other names.
    • Fix crash from old-style file. [Bug 392054].
    • Fix cell contents reading from old v7 files. [Bug 392054].
    • Make sure we call g_thread_init early. [Bug 392050].
    • Fix CRITICALs with DOLLAR.
    • Fix progress display for .gnumeric and thus splash. [Bug 392546].
    • Don't allow "1D2" [win32] or "0x1234p1" as numbers. [Bug 393347].
    • Fix redraw problem. [Bug 393359].
    • Don't complain over encoding-less documents unless there is a reason.
    • Don't pull in popt. [Bug 396329].
    • Fix OO import crash. [Bug 396200].
    • Fix part of problem loading style conditions. [Bug 397679].
    • Fix generated libspreadsheet-1.7.pc [Bug 399550].
    • Fix tsv import problem. [Bug 404264].
    • Fix R1C1 crash. [Bug 407274].
    • Fix R1C1 issues. [Bug 407247].

Goffice 0.3.6 aka "Quiet in the Library" is now available.

  • Jody

  • Morten

    • Simplify format widget a bit.
    • Implement go_strtod and go_strtold.
    • Fix marshalling of pango formats with strikethrough set.
    • Make the GOFormat type private.
    • Extract go-locale.[ch] from go-format.[ch].
    • Fix UMR.
