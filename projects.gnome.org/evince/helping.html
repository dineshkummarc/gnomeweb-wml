Helping with Evince

Triage Bugzilla

Use this link for looking at NEW bugs with Evince. Please make sure the new and unconfirmed bugs conform to the bug reporting guidelines laid out in the testing section. You may want to look at the development bug list for alternative bugzilla queries.

The GNOME Bugsquad has a good triage guide for people new to bugzilla and GNOME. Read up on that before doing any triage work with bugzilla and if you know that please also read our bug reporting guidelines in the testing section.

Distribution Support

Evince needs as much testing and widespread distribution as possible in order to satisfy its goal of becoming the default document viewer in GNOME and replace the craziness that exists.

Ask your favorite distribution to package Evince and use it at your default document viewer.

Bugs with PDF

This section has been moved to Evince Poppler Bugs on the wiki site

Often times a bug filed against Evince turns out to be a bug in the PDF backend that Evince uses, called Poppler. If a bug is identified as being a bug in the PDF backend, Poppler, we need to then file that bug against the Poppler bugzilla. To help out you'll need to follow these steps:

  1. A bug filed is determined to be against the PDF backend
  2. An Evince developer should link to this page and request that the reporter follow these instructions for moving the bug to Poppler
  3. The reporter searches poppler bugzilla for similar bugs
  4. If no similar bugs are found the reporter opens a new bug against Poppler
  5. The new bug should contain the original report as well as the last couple comments that are most relative to the poppler backend, plus provide a link back to the Evince bug you are copying from.
  6. Add a link to the new Poppler bug in the original Evince bug
  7. Finally close the Evince bug (or ask an Evince hacker) NOTGNOME
