Requirements for GNOME Accessibility:
 

  • Offer maximum functionality "out of the box"
  • Provide API for apps to provide additional UI info
  • Be interoperable with non-GTK+ applications
  • Interoperate with other accessibility frameworks (Java, Mozilla)
  • Provide flexible "out-of-process" API for ATs
  • Be platform agnostic
  • Allow AT operation without offscreen model
  • Be deliverable in 2.0 timeframe
  • Be extensible
  • Minimize performance impact on existing apps
  • Be easy to use by application maintainers and ATs
