Extensibility:

  • new interface types may be compatibly added
  • inheritance/factory model makes it easy to add ATK implementations 

    • incrementally for new widget types
  • non-toolkit-specific API allows extension to other toolkits

    • (java,bridge shipping with Sun's GNOME 2 release); Netscape; Motif; KDE;Mac)
  • API approach allows porting to other environments (i.e. embedded, etc.) in the future.
  • Standard CORBA protocol gives network transparency, portability; can work in a heterogeneous environment.
