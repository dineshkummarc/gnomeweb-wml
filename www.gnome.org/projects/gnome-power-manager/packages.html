Download

It is advisable to try the pre-compiled package for your distribution before you try to make GNOME Power Manager from source. If you want to add a link to a new distro, please drop me an email with the link.

Fedora Core (5 and above) has the gnome-power-manager package in core and it should be installed by default. If it is not installed, just do: 

yum -y install gnome-power-manager

Ubuntu Dapper and Edgy have the gnome-power-manager package in main and it should be installed by default. If it is not installed, just do: 

apt-get -y install gnome-power-manager

Package by Ken Vandine for Forsight Linux

Package by James Ogley for SUSE Linux

To use this APT repo, please follow the instructions on the usr-local-bin.org website. Then you can use the following commands: 

apt-get update
apt-get upgrade
apt-get install gnome-power-manager

Compile notes for Arch Linux

If you try to compile the tarball with Arch Linux, you may get the error 

libtool: link: cannot find the library `/usr/lib/libgobject-2.0.la'

GNOME Power Manager uses pkgconfig (like most open-source projects, but Arch Linux has a bug where libnotify.la references other files. The .la files are soon to be removed from Arch Linux, and if you manually delete the file libnotify.la, the tarball will compile.

An ebuild is available in portage. You know what to do.

debs by Alejandro Vera for Debian Linux

CVS

You can check out the latest GNOME Power Manager code using CVS. You can use the viewcvs interface too. 

export CVSROOT=":pserver:anonymous@anoncvs.gnome.org:/cvs/gnome"
cvs -z3 checkout gnome-power-manager

Compiling The Code

To compile and make, I use the following procedure: 

cd gnome-power-manager
./autogen.sh --prefix=/home/hughsie/root --with-gconf-source=xml::/home/hughsie/.gconf
make
make install

Downloads

GNOME Power Manager should come bundled with most modern distributions.

The latest source releases can always be found at the GNOME ftp site.

Make sure to get latest release of HAL, the service that GNOME Power Manager depends on.


Feedback

Bugs should be reported to the GNOME Bug Tracking System.

Please read the FAQ and the bug reporting guidelines before creating a new bug.

You can get in touch with developers and other users by sending a mail to the Power Manager mailing list or search the [archives].
