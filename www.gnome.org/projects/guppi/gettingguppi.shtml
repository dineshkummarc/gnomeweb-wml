Getting Guppi

The current development release of Guppi is 0.40.0, released on October 18, 2001. There has not yet been a stable version of Guppi released.

Getting Source From Tarballs

You can download the a tarball containing the Guppi source code from here.

Getting Binaries From RPMs

Guppi RPMs are available for Red Hat. These may or may not work with other RPM-based distributions.

There is sometimes a brief time interval between the release of a new Guppi tarball and the appearance of the updated RPM.

Getting Binaries From A Debian Package

Guppi is currently being packaged as part of Debian Woody.

Getting Source From CVS

The current incarnation of Guppi is in the module guppi3 on the GNOME CVS server. You can learn about using CVS here. (Do not check out the modules guppi or guppi2 --- these contain old, crufty, obsolete code.)

You can browse the Guppi source code here.

The usual caveats about code from CVS repositories apply: it may not work, it may not even compile. I try to avoid checking in code that is overly broken, but I can't make any promises.

Building Guppi From Source Code

If you are building Guppi from CVS, you first need to run ./autogen.sh in the guppi3 directory. Otherwise, the standard ./configure and make all should work.
