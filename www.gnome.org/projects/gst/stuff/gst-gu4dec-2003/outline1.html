    What are the GST?

    • Advantages
      • They are user friendly
      • They are cross-platform
      • They are possibly cross-desktop
      • Abstracts the user from unneccesary concepts
      • They are integrated in the GNOME Desktop
      • They add consistency to the desktop

    What are the GST? (2)

    • Disadvantages
      • There is added complexity in the tools' structure
      • There are important tools missing
      • There are important supported systems missing

    GST in relation with the other tools

    • Since the beginning of times, distros have been forking some of their internals
    • The other tools only work in the system they are designed to work
    • Reflect lack of consistency between tools

    Why the BE/FE structure?

    • The GNOME System Tools are formed by:
      • A backend
      • A frontend

    GST Structure

    Steps followed by the GST

    • The backend extracts the information and stores it in a hash table
    • The hash table is translated to XML and sent to the frontend
    • The frontend interprets the XML and shows the data to the user
    • When the user changes anything, these changes affect the xml
    • When the user hits 'apply', the changed XML is sent back to the backend
    • The backend translates the XML to his hash equivalent
    • The backend applies the changes to the system files

    Advantages of the structure

    • Allows the creation of several frontends using the same single backend
    • Could allow remote configuration
    • The XML is common through all supported systems
    • Provides a platform that eases the development of other tools

    Backend: Why PERL?

    • Advantages
      • PERL was thought to handle text files, and all of the system files are text files too
      • Regular expressions are really powerful when extracting data from files or from commands' output

    Backend: Why PERL? (2)

    • Disadvantages:
      • It's an scripting language
      • There is very few people who know/like PERL for programming

    The future of GST

    • Adding more tools, such as: printers, X server, fstab, cron, firewall, ...
    • More sophisticated authentication method
    • Remote configuration
    • Possibly removing PERL from the backends (maybe C+Flex?)
    • More and more integration with the desktop
