Getting From CVS

To get the very latest copy of Evince you'll need to pull it from CVS. Here's how you'll do it. 

	export CVSROOT=':pserver:anonymous@anoncvs.gnome.org:/cvs/gnome'
	cvs login
	[press enter for blank password]
	cvs -z3 co evince
	cd evince

Getting From Package

Download the latest source package from the Evince package directory: http://ftp.gnome.org/pub/GNOME/sources/evince/. Then untar the files like this: 

	tar -zxvf evince-0.1.0.tar.gz
	cd evince-0.1.0

Building From Source

Assuming you've changed into the Evince source directory from either a package or CVS you now want to start the build. 

	./autogen.sh --prefix=/usr/local
	make
	make install (sudo make install may work for you)

Testing

Now that you have Evince installed try it out on different document types, if you find a problem with the thumbnailing, indexing, or rendering of the PDF or Postcript please file a bug.

Reporting Bugs

When reporting bugs about Evince it is most helpful to include the document you were viewing. If you can provide a link to this document please include that link so the Evince developers can test against that document as well.

Report bugs for evince to GNOME Bugzilla under the Evince product.

Ideal Bug Report

Summary: Crashed whlie opening PDF document

Description: I clicked on a link from Epiphany which opened Evince for viewing a PDF document. Evince crashed before actually rendering anything at all.

Here is the link to the document: http://link.to.document/doc.pdf

Below is the backtrace of the crash: [begin backtrace]

If Evince crashes include a backtrace, use this Getting Traces Guide for help.

If Evince doesn't thumbnail or index or render properly please indicate the type of document you were viewing and if at all possible include a link to the document. Try not to attach your document to the bug as we don't really need all your documents sitting in the bugzilla database.
