Gnumeric 1.7.12 aka "Rocks for Rob, Roses for Harlan" is now available.

This development release fixes some long-outstanding named range issues and improves XLSX import and export.

The Windows support also got some nice fixes, but we still do not have a build. We realize that we have to provide a build eventually, but so far Windows volunteers with build expertise have been almost totally absent.

This release requires the concurrently released Goffice 0.5.0 and Libgsf 1.14.6
