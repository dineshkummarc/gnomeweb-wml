Help with the development of Straw is welcome.

If you want to contribute code, please see the TODO list included in the sources (not that you can't hack on anything you want), and post in the mailing list first before starting to hack on something. This way we can avoid duplication of effort.

Reports

Bug reports and request for enhancements are always welcome. We are currently using two sites for bug reports: one in GNOME Bugzilla and one here in Savannah. We encourage bug reports to go to GNOME Bugzilla.

Translations

Straw is internationalized, and translations are very much welcome. Translators are encouraged to cooperate with the GNOME Translation Project. Additional information can also be found at the Translation Project page. The README also contains hints and tips on how to use setup.py (distutils) to build and help you with the translations.

Development

Arch/TLA

Current mainline: juri@iki.fi--2004a/straw--mainline--0.26
Location: http://www.helsinki.fi/~pakaste/arch/2004a

To get started:

  1. Create your archive (if you don't have one yet)

    tla make-archive myname@mydomain.com--2004 /home/me/path/to/archive

  2. Create your own branch.

    tla archive-setup myname@mydomain.com--2004/straw--mainline--0.25

  3. Register the main Straw archive (If you haven't done it yet)

    tla register-archive juri@iki.fi--2004a http://www.helsinki.fi/~pakaste/arch/2004a

  4. Tag your archive from the main Straw archive

    tla tag juri@iki.fi--2004a/straw--mainline--0.25 myname@mydomain.com--2004/straw--mainline--0.25

  5. Get the sources and save it to your preferred directory

    tla get myname@mydomain.com--2004/straw--mainline--0.25 straw

For more information about Arch/TLA, visit http://www.gnuarch.org.

CVS

To get the CVS version, just follow the steps in http://developer.gnome.org/tools/cvs.html. The module is called straw.
