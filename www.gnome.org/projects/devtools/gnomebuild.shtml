GNOME Build

Lead Developer: JP Rosevear

gnome-build will provide the project management components of the development environment. This includes management of source-code resources, generation of makefiles, and build management.

We have the following goals for gnome-build:

  • Backend independent gnome-build will provide support for multiple build systems to best integrate with the user's existing systems.
  • Autoconf/automake integration Our main focus will go into an autoconf/automake backend. This backend will create and read Makefile.am and configure.in files in a manner consistent with the majority of current free software projects.
  • Configuration support Multiple configurations will be supported by the gnome-build framework. For example, a user could seperate Debug and Release configurations with different build flags between them. You find the current version of GNOME Build in gnome CVS.
