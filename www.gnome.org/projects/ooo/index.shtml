Gnome technologies used

While OO.o didn't start it's life as a Gnome project, it is increasingly deeply embedded into Gnome - consider what parts of Gnome and the freedesktop are used in G/OO.o:

.
  • Gnome-VFS - G/OO.o allows transparent access to allow documents to be loaded / saved from a range of Gnome VFS backends, eg. Samba shares.
  • fontconfig - G/OO.o shares the same font list as Gnome resulting in less user confusion.
  • Gtk+ - G/OO.o uses Gtk+ theme colors, and font settings.
  • libart - this is used to do polygon clipping for printing complex gradient filled polygons, and for MS import/export.
  • gnome-cups
  • recent-files - this allows recently used files to show up in the System, Recent Files menu in the panel.
  • startup-notification

Ongoing work

There is lots of ongoing work on G/OO.o increasing quality and integration of OO.o on Unix and as a Gnome application

  • Gtk+ theming - work is underway on this, and should land in G/OO.o shortly see Dan's work.
  • Gtk+ mainloop - work to convert VCL to use gtk+ and the glib main event loop is working well in OO.o CVS but is unfortunately invisible.
  • Bonobo component - OO.o is available as a bonobo component, though ideally needs the gtk+ plug/socket (XEmbed) to be really usable, see Martin's work.
