Development

Old rocking technologies

Gnome Scan is designed and implemented in order to fit Gnome plateform. Gnome Scan is implemented in C/GObject in order to provide easy bindings to high level languages like C++, python, ruby, C#, … and why not a vapidef ?! This will allow to integrate Gnome Scan in as many application as possible.

The choice of writing Gnome Scan from scratch instead of forking XSane doesn't mean that Gnome Scan reinvent the wheel. Gnome Scan use well known technologies like SANE, the free scanner access library which include wide device support as well as regular Gnome technologies like Gtk+, Glib/GObject for bindings, la POO, les threads, …

Gnome Scan is well documented using Gtk-Doc. Documentation even include a widget gallery like Gtk+ documentation ! We lake tutorial, you might want to produce one, as well as user manual.

Leading edge technologies

Gnome Scan is build on top of leading technologies from free desktops. Scanner detection and hotplug through HAL, inter-process communication with DBus, etc. Scanner buttons support with new HAL addon, etc.

Image post processing is done through the fabulouse Gegl library, a project from The Gimp developers. Gegl implements already tons of image processing operation: rotation, denoise, sharpen, enlight, darken, etc. With Gnome Scan, new scan specific operation will land such as OCR, deskew, etc.
