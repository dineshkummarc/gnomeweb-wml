Contributing

GtkSourceView development relies on voluntary contributions and everyone is invited to help.

If you are interested in helping us to develop GtkSourceView, please contact the developers on IRC and/or send a message to the GtkSourceView mailing list and offer your help.

Code Repository

GtkSourceView is hosted on the GNOME Subversion repository in the gtksourceview module. You can check out the latest development version with the following command: 

svn co http://svn.gnome.org/svn/gtksourceview/trunk gtksourceview

For more information about Subversion, see the instructions on how to use the repository.

See the HACKING file for information about the GtkSourceView coding style and Subversion usage guidelines.

Bug reporting

Bugs should be reported to the GNOME bug tracking system, product gtksourceview.

You will need an email address to register (if you haven't already) before you can use the system to file a new bug report.

There are guidelines for reporting new bugs, try to make sure you follow them, it helps us help you much more effectively. The use of keywords also helps us prioritize bugs.

Patches

Patches should be submitted to the bug tracking system.

If the patch fixes a known bug, add the patch as an attachment to the corresponding bug report. Otherwise, enter a new bug report that describes the problem the patch fixes, and attach it to that bug report.

Patches should be in unified diff form (use the -up options of GNU diff) and should follow the coding style described in the HACKING file.

See the Submitting Patches page on the GNOME Wiki for more information on how to submit a patch.

Mailing lists

You can subscribe to the GtkSourceView mailing list, or change your existing subscription, visiting the gnome-devtools info page.

To post a message to all the list members, send an email to gnome-devtools (at) gnome org.

To see the collection of prior postings to the list, visit the gnome-devtools archives.

Internet Relay Chat

You can find the developers on #gedit channel at irc.gnome.org.

Wiki

We have a Wiki for drafting, designing and proposing GtkSourceView development available at http://live.gnome.org/GtkSourceView.
