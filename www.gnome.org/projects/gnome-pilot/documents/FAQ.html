    Q: What is gnome-pilot ?

    A: A package aiming at offering as complete as possible integration between your
        PalmPilot pda and the Gnome desktop. This means a daemon for monitoring Sync
        requests, backup and file installation capabilities and conduits for Gnome
        applications.

    Q: Where can I get it ?

    A: Newest releases are available at  http://www.gnome.org/gnome-pilot/download .
        The newest development code is also available via anonymous CVS from the
        Gnome  CVS repository. See  http://developer.gnome.org/tools/cvs.html .

    Q: GOB_CHECK missing ?

    A: GOB is the Gtk+ Object builder, that builds .c and .h files from .gob files. You can find it
        http://www.5z.com/jirka/gob.html.

    Q: I've got a XXX device, does that work with gnome-pilot ?

    A: gnome-pilot should work with anything that pilot-link works with, as the underlying
        communication is done by pilot-link.
    A: As for Visors with USB support, check  http://usbvisor.sourceforge.net/  for info on
        connecting the Visor to Linux via USB.

    Q: I get "*** ERROR *** Cannot connect to GnomePilot Daemons", huh ?

    A: If you compiled gnome-pilot yourself, you probably misconfigured your compile.
        More specifically, the --sysconfdir parameter to configure was most likely omitted.
        Try the following :
    $./autogen.sh --prefix=`gnome-config --prefix` \
      -sysconfdir=`gnome-config --sysconfdir`
        This should ensure, that the .gnorba files end up where the CORBA nameserver can find them.
        If you did not compile gnome-pilot yourself, check manually that the gpilotd.gnorba file is placed
        along the other .gnorba files. It should be in the same directory as eg. panel.gnorba.

    Q: All syncs end with timeout or SIGINT ?

    A: This bug is known, no need to bug report it. However, this is a very odd bug that has yet not
    been fixed (0.1.47 at the time of writing). The bug only appears if gpilotd is started as a CORBA
    service, eg.  by starting pilot_applet before gpilotd. If you manually (or pr. config) start gpilotd before
    other gnome-pilot tools, you should be safe and sound, if not, please let me know that this also fails.

    The bug is sporadic, and may appear at one time, and after watching a South Park episode be gone
    again. Also, this bug is apparantly related to the serial port, as you should no experience it with
    simulators such as XCopilot.

    UPDATE: 20000114: This bug should now be fixed, as of version 0.1.48.

    Q: What is a SyncId ?

    A: The syncid is just a random number, that identifies your machine. The pilot uses this number during
    sync to check if the last synchronize was with the same machine as the current. Thus this number only
    has to be unique within the set of machines with which you sync. In most cases, you won't have to set
    this value.

    Q: What is "Synchronize Action" ?

    A: The synchronize action determines how conduits should transfer data between the local storage
    and the handheld's storage. This setting can be used to impose a certain action on the conduits.
 

Use conduit Settings

 

 Use whatever the conduit is configured to do. 
Synchronize Call synchronize. This synchronizes db's at the record level. See note below.
Copy from pilot Call copy from pilot. The deletes the local storage, and then copies all records from the pilot. See note below.
Copy to pilot As copy from pilot, only the other way around. See note below.
Merge from pilot Perform record level merge but do not modify/add records on the pilot.
Merge to pilot Perform record level merge but do not modify/add records in the local db.

    Note : If the imposed action is not supported by the conduit, no synchronization will happen. Also
    please note, that copy to/from pilot deletes db's before proceeding. This can cause dataloss is the
    synchronization is ended prematurely.

    In most cases, "Use conduit settings" will be the option you want.

    Additional Note: This option may be removed.

    Q: What is the "User Id" ?

    A: The user id identifies your pilot. In multi user environment (when gpilotd truly supports that), it
    will be used to match the pilot with you. When you buy or reset your pilot, it has a ID=0. Normally, a
    pilot with ID=0 will mean just that, a blank slate. Therefore, you should not use 0 as an ID.

    If you have used the 3Com supplied software for Windows/Mac, the pilot will have an ID set.
    Other sync packages might also set an ID, in this case, try the "get from pilot" button in the
    Pilot-Link capplet. If this results in a ID=0, please set it to another value, and press
    "send to pilot". Not doing this, will someday bring trouble (we're talking about stuff like missing
    toilet paper and worse!)

    Q: How do I install files ?/How does the file conduit work ?

    A: To install files, first, you must remember to enable the file conduit. If this conduit is not enabled,
    the installation will not happen. Once enabled, there are three ways :

  1. Rightclick on prc/pdb files in GMC, there should be an option called Install Now and Install Later. Now displays a box asking you to press sync, Later queues the files till a later sync.
  2. Drag and drop files onto the panel applet.
  3. Run gpilot-install-file [-l|-n] [FILES] from commandline. The -l corresponds to the Install Later in GMC, -n to Install Now.

    Q. When I delete or edit items in gnomecard, the change is not reflected on the pilot?

    A: This is limitation of current gnomecard program. There's no way for the address conduit to find
    out, which entries have been modified or deleted. Changes in the gnomecard implementation will be
    required to add such functionality.

    Q. What are these strange garbage characters in my gcrd file?

    A: Certain characters need to be quoted to correctly read/write vcard files. "=A0" represents "\n",
    "=3D" represents "=" etc., "=" by itself at the end of the line means the entry is continued on the
    next line.

    Q: My AvantGo doens't start/work, que ?

    A: It has been reported, that eg. the Minstrel wireless modem which apparently adds AvantGo to the
        flash does not work, since the main DB is not reported (since it's in flash). The temporary
        fix (making the conduit start if another AvantGo base is present) also failed. Stay tuned
        for updates.

   Q: Hot do I install periodically updated local DB's ?

    A: If you eg. have a cron job that updates some .pdb's locally, let the cronjob call
        gpilot-install-file -l file1.pdb file2.pdb ...
    that should do the trick.
