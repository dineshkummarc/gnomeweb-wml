Epiphany Frequently Asked Questions

1. General Information

1.1 Why another GNOME browser, what about Galeon?

Epiphany was forked from Galeon to be able to build a more usable browser for non technical users. This involves removing confusing/technical features but more importantly redesign and add new features important for this part of the users. (Marco Pesenti Gritti)

1.2 Why is this project named "Epiphany"? There is already another project with that name!

The maintainers of the other project (a boulder-dash clone) agreed to share the name. In any case of conflict, epiphany-browser is used instead.

1.3 What about bookmark management?

The traditional way of organizing bookmarks seems to be to complicated for the average user. Assigning topics to bookmarks instead of putting them in hierarchical folders provides a more intuitive way for managing bookmarks. This causes a few differences to traditional bookmarks systems:

  • For now, it is not possible to organize bookmarks in a multiple level hierarchy.
  • Bookmarks could be assotiated with multiple topics.
  • The recommended way to access bookmarks is the bookmark dialog, not the bookmarks menu.

2. Installation

2.1 What's the recomended way to compile Mozilla?

The suggested way for compiling Mozilla is

./configure --prefix=YOUR_GNOME_PREFIX
  --enable-default-toolkit=gtk2 --disable-mailnews --disable-ldap
  --disable-debug --enable-optimize --disable-tests --enable-crypto
  --enable-xft --with-system-zlib
make
make install

2.2 How can I disable compile warnings?

Add the following flag when you ./configure ephy: --enable-compile-warnings=no.

3. Configuration

3.1 Where can I change proxy settings?

Epiphany uses the GNOME default proxies. They can be modified (session wide) by using gnome-network-preferences. The corresponding gconf keys are /system/http_proxy/host, /system/http_proxy/port and /system/http_proxy/use_http_proxy.

3.2 Can epiphany load the pasted url on middle-click like mozilla?

Use gconf-editor to enable it. The key is /apps/epiphany/general/middle_click_open_url.

4. Troubleshooting

4.1 Epiphany doesn't start but shows a dialog “Epiphany can't be used now”.

Try running the command bonobo-slay from the console. This may fix the problem.

4.2 Where is the spinner / the epiphany icon?

Both are part of the default GNOME icon theme. So make sure you have a recent gnome-icon-theme package installed; you need version 1.0.6 or later. If the problem persists even then, it is a bug in your theme, not in epiphany.

4.3 Epiphany doesn't load the plugins I have installed!

Some distros install plugins in /usr/lib/mozilla/plugins (instead of /usr/lib/mozilla-version/plugins), for epiphany to catch you have to set the MOZ_PLUGIN_PATH environment variable to the plugin directory.

4.4 Epiphany doesn't open my favourite email application when I click a mailto: link!

You should assign a handler (evolution, kmail, mozilla mail, etc) for the mailto: format using the Advanced Desktop Preferences, "file types and programs". You can check the existing handlers for other protocols/services in that configuration program for the exact syntax.

4.5 Is there a way to close all open windows at once?

No.

4.7 Where's the SSL certificate manager in Epiphany?

If you have the Certificates extension from Epiphany Extensions package installed, you can access the certificates manager from Tools->Manage Certificates. Alternatively, you can just type the url chrome://pippki/content/certManager.xul in the location entry in Epiphany.
