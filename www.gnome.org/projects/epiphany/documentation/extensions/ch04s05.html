4.5. Connecting Signals

GLib has a very flexible signals system. Use it. The impl_attach_window function has an EphyWindow object. For example, the tab_added signal will fire whenever a tab is added to that window. Put the following in impl_attach_window to hook the tab_added signal to your own callback:

  GtkWidget *notebook;

  notebook = ephy_window_get_notebook (window);

  g_signal_connect_after (notebook, "tab_added", G_CALLBACK (tab_added_cb), extension);
  g_signal_connect_after (notebook, "tab_removed", G_CALLBACK (tab_removed_cb), extension);

tab_added_cb can connect signals of its own. Of particular interest are the EphyEmbed signals (prefixed by ge_. The signals are listed in epiphany/embed/ephy-embed.c. For example, to call a function when the browser changes to a new location:

static void
location_cb (EphyEmbed *embed, char *address, EphyTab *tab)
{
  /* Actions to perform on location change */
}

static void
tab_added_cb (GtkWidget *notebook, EphyEmbed *embed, EphyFooExtension *extension)
{
  EphyTab *tab;

  tab = EPHY_TAB (g_object_get_data (G_OBJECT (embed), "EphyTab"));
  g_return_if_fail (EPHY_IS_TAB (tab));

  g_signal_connect (G_OBJECT (embed), "ge_location", G_CALLBACK (location_cb), tab);
}
Warning

Free all memory and disconnect all signals when they're not needed. In the case of this ge_location signal connection, it should be disconnected when the tab is removed:

static void
tab_removed_cb (GtkWidget *notebook, EphyEmbed *embed, EphyFooExtension *extension)
{
  EphyTab *tab;

  tab = EPHY_TAB (g_object_get_data (G_OBJECT (embed), "EphyTab"));
  g_return_if_fail (EPHY_IS_TAB (tab));

  g_signal_handlers_disconnect_by_func
    (embed, G_CALLBACK (location_cb), tab);
}

Likewise, clean up in impl_detach_window. In our example:

  GtkWidget *notebook;

  notebook = ephy_window_get_notebook (window);

  g_signal_handlers_disconnect_by_func
    (notebook, G_CALLBACK (tab_added_cb), extension);
  g_signal_handlers_disconnect_by_func
    (notebook, G_CALLBACK (tab_removed_cb), extension);
