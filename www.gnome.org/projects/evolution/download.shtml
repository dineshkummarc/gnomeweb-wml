Already Included

Quick and Easy

Red Carpet, a software management solution from Novell, enables users to intuitively install software.

Your distribution may be supported in Red Carpet's distribution-specific channels. If so, installing Evolution is just a few clicks away (after installing Red Carpet*).

* Red Carpet is included with Novell Linux Desktop and SUSE Linux — and available standalone as well.

Red Carpet - Snapshots

An alternate method—for the risk-takers out there—is to use Evolution snapshots available through public servers in the evolution-snapshot and ximian-connector-snapshot channels.

Unstable developer snapshots are also available in the evolution-devel-snapshot.

Grab the Source From CVS

CVS stands for Concurrent Versioning System, and is a common way of storing source files (and revisions) in the Open Source community. It enables developers to track the changes between versions and empowers teams to work on the same codebase together. It is also a way for some power users to check out the latest happenings in a software project.

Anonymous CVS Checkout

Checking out Evolution's source from anonymous CVS is actually pretty easy. It can be done with two commands (listed below).

cvs -d ':pserver:anonymous@anoncvs.gnome.org:/cvs/gnome' login
cvs -z3 -d ':pserver:anonymous@anoncvs.gnome.org:/cvs/gnome' checkout evolution

To check out Evolution from CVS, first change to a user-writable directory you would like the source directory to be contained within. You will then want to paste the lines listed above, one at a time. After the first line is pasted (and enter is pressed), a login prompt will appear. When the prompt arrives, simply hit your enter key again and you will be returned to your shell's prompt. At this point in time, paste the second CVS line above and press enter once more.

Provided that everything checked out correctly, you can then type "cd evolution" to change to the source directory, then "./autogen.sh" to generate (and run) your ./configure file. After that, you should be able to do "make" (and wait a bit), then (as root) "make install".

Later on, if you would like to update your source to the most recent version, please perform the following command (in your evolution directory).

cvs -z3 update

This page just has the basics of CVS listed above. If you'd like more information, please consult the official GNOME developer's guide to CVS.

GNOME Developer CVS Checkout

For GNOME developers, the methods above are essentially the same. The only changes are the initial CVS strings, as the server, server type, and usernames change.

cvs -d ':ext:<username>@cvs.gnome.org:/cvs/gnome' login
cvs -z3 -d ':ext:<username>@cvs.gnome.org:/cvs/gnome' checkout evolution

Often times, the following (if your username is the same as your developer username on the CVS server is the same) will work.

cvs -d ":ext:${USER}@cvs.gnome.org:/cvs/gnome" login
cvs -z3 -d ":ext:${USER}@cvs.gnome.org:/cvs/gnome" checkout evolution

For more information, please consult the anonymous CVS checkout section directly above, or consult the official GNOME developer's guide to CVS.
