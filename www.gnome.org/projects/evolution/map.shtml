Quality Targets

Evolution 2.0 should have a total of 0 P1-P3 bugs and less than 40 P4 bugs. Evolution Data Server 1.0.0 should have 0 P1-P3 and less than 10 P4 bugs. Evolution Data Server 1.2.0 should have 0 P1-P4 bugs.

Evolution 2.0 and Evolution Data Server 1.2 UI Freeze

UI changes prior to the UI freeze will be should be limited to:

  • spelling mistakes
  • focus issues
  • Human Interface Guideline improvements
  • Usability improvements from the UI designer

After the UI freeze, UI changes other than those listed above would have to be related to a P1-P3 bug and approved by JP and the product design team.

Evolution Data Server 1.0 UI Freeze

As noted above, the GNOME 2.6 UI freeze is already in effect and the GNOME 2.6 guidelines will be followed.

Evolution 2.0 and Evolution Data Server 1.2 String Freeze

After the string freeze the only string fixes that can be made would be:
  • spelling mistakes
  • string changes to make translation easier

Any string changes other than those listed above would have to be related to a P1-P3 bug and approved by JP.
