Developers informations

Getting Involved

To get involved, contact maintainer gFooBarMaintainer

More information about getting involved with other GNOME apps (and many tips that apply to gFooBar) can be found at http://live.gnome.org/GettingInvolved


Debugging

In most ways, gFooBar is a typical GNOME App; you should refer to useful stuff that applies to gFooBar

Translating

Just as many other GNOME applications, gFooBar is translated by the GNOME translation team.


Drawing Icons

All icons drawn for gFooBar should match the GNOME Icon Style Guide


Documentation

If you want to write documentation for gFooBar, you should join the GNOME Documenation team.


Downloads

gFooBar should come bundled with your distribution.

The latest releases can always be found at the GNOME ftp site.

The latest stable release is version (changelog).


Feedback

Bugs should be reported to the GNOME Bug Tracking System.

You can get in touch with developers and other users by sending a mail to the nautilus mailing list [Archives] or by joining the #gFooBar channel on irc.gnome.org.
