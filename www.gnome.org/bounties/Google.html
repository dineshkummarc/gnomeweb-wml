Improve login time (Bug #306042) | | Discuss | Claim Bounty
Optimization Bounty: $4500
The time it takes a user to login to GNOME has not changed very much in the last five years, despite massive improvements in hardware performance. Some analysis has shown that login time is bounded by a heavy I/O load. Havoc Pennington has pointed out a 3-4 second delay caused by GConf initialization.

This bounty will go to the person (or team) who submits a maintainer-accepted patch which results in the single largest reduction in login time.

Reduce Evolution memory usage (Bug #306043) | | Discuss | Claim Bounty
Optimization Bounty: $4500
This bounty will go to the person who submits the best accepted patch to Evolution which results in the greatest reduction of run-time memory usage under ordinary usage. Memory reductions to the mailer, addressbook and calendar will all be considered.
Global memory analysis tool (Bug #306044) | | Discuss | Claim Bounty
Optimization Bounty: $4500
Global memory analysis tool. A tool that lets you look at the memory usage of all running GNOME programs and help account for where the memory is going. Must properly account for sharing of memory between processes.
Use a mmap'able cache for fontconfig (Bug #169345) | | Discuss | Claim Bounty
Optimization Bounty: $4500

On the startup of every program, fontconfig allocates about 100 kb of data. This data should be read from a memory mappable cache on disk. This way, the data is shared between processes. Also, it allows more expensive techniques to optimize the data storage.

A solution to this bounty will:

  • Create a program that generates an on-disk cache.
  • Modify fontconfig to read from this file, allowing a fallback path with the existing code.
Live documentation editing
Implement a live web-based wiki-like editor that allows generation of content that can be rolled back into our documentation. Allows users to contribute annotations and additions to documentation (e.g. can add documentation to an undocumented function), save it. Other people visiting the page will see your additions to the documentation, and the maintainer of the module can extract the addition and trivially roll it into the official documentation source.
Gnome architecture overview
Write a document giving an overview of the GNOME development platform's architecture. The target audience for this document includes developers who want to begin developing software on GNOME, people who want to begin contributing to GNOME, or system administrators who want a better understanding of how the pieces fit together. This bounty will go to the authors of the best paper.
library.gnome.org
Implement "library.gnome.org" which aggregates documentation from releases of GNOME platform libraries into a searchable, readable, consultable form.

Have an idea you think no one's ever had before? An innovative application that doesn't exist but that everyone needs? Something so cool it just has to be done?

This bounty will be awarded to the authors of the most unprecedented application. Submissions will be judged by a panel appointed by the GNOME board of directors.

Panel applet bundle system
A framework that lets developers implement panel extensions as simple to write single file bundles. Bundles will require no native code, probably written in Python (so they work between library versions etc), and will be user installable from web pages. Must include a tool for building the bundles.
Group mails by conversations in Evolution
Gmail provides an innovative new way of reading your emails: as conversations wherein the mails you send and the mails you receive in a given thread are all visible once, in chronological order. Implement this in Evolution.
Paste by example
Paste by example is a feature that allows users to learn "by example" the features of an office suite. It will let people learn to create any formatting or object seen in a document sent to them (or one they created 3 years ago). Office objects in the clipboard buffer (e.g. a graph or a formatted sentence) can be pasted into the current document, but step by step as a dynamically generated tutorial (showing the dialogues and commands users could have used to recreate the object in question). Feature can be implemented in OpenOffice.org or AbiWord.
Search party
Search party is a feature that allows adhoc chat rooms to form based on what people are searching for on the web (using e.g. Google). Implement the search party web browser extension for Firefox or Epiphany. Define the search party client/server "protocol" and implement a server.
Shared music spaces
Shared music spaces allow people in a shared chat room (e.g. an IM chat room) to listen to a music stream playing on any of their computers. Feature will be added using Rhythmbox and Gaim.
Conference content
Develop a complete set of content for shows. This includes posters, imagery, and brochures targeted toward different purposes (prospective developers, trade shows, corporate sponsors, etc)
