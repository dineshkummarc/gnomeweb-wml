Known Issues

All software, when it is released, contains bugs the developers know about but have elected, for a variety of reasons, not to fix before releasing. Free software is no different in this regard from proprietary software, except that with free software, we tell users about them.

We also encourage our users to report bugs so that they can be fixed. The best way to report bugs found in GNOME 2 is to use the Simple Bug Guide. The Simple Bug Guide will take you through the necessary steps to file a quality bug report, and make sure that it is tagged appropriately. If you're too advanced for anything with the word 'simple' in it, there is also the traditional bug form. Please make sure to add the 'GNOME2' keyword to any bugs reported that way.

More details on bugs already reported can be found at our Bugzilla. Among the most prominent GNOME 2 bugs:

NFS-mounted $HOME directory

For people with NFS-mounted $HOME directories, you cannot drag files from the Nautilus view windows to the trashcan (you get an error about the trash file not being a directory). You can still delete files by selecting and then pressing the Delete key, or by context-clicking on the item in question and selecting Send to Trash. This will be fixed in the next release, and is bug 82644.

.gnome

GNOME 2 uses $HOME/.gnome2 and $HOME/.gnome2_private as directories for storing stuff. However, you cannot completely remove $HOME/.gnome, since gnome-vfs needs it and does not create it, so any modifications that gnome-vfs needs to save in the user's directory will not be saved. Thus, users need to create a $HOME/.gnome directory before starting GNOME 2. This will particularly affect people doing an install on a clean system or those who have removed .gnome to get around some upgrade problems. This will be fixed in 2.0.1 and is bug 84183.

Cannot drag and drop from Nautilus list view

You can't drag and drop files from the list view as you can in the icon view. It is hoped that this will be fixed in the next release. You can track progress on this bug or contribute yourself via bug 74974.

Menu editing is incomplete

Because a number of bugs existed in the menu editing implementation, and the best fixes for some of them may not be reverse compatible, it was decided to remove most menu editing functionality in this release. It will be fixed and re-enabled in the 2.0.1 release. In the meantime, a number of related bugs are filed, including 73770, 73773, 81670, and 82129. Details on the menu architecture and tips for manual editing of menus can be found here.

new mime type handlers are not always visible in nautilus

Mime-type handling in GNOME 2 is generally much improved over GNOME1.4. However, when a new mime-type handler is installed in GNOME 2, it does not show up correctly in Nautilus right-click menus. It is hoped that this will be fixed in GNOME 2.0.1. Track it at bug 75620.

Internationalization of gnome-terminal is broken

Unfortunately, gnome-terminal is broken in many non-ASCII locales. A fix for this was proposed but caused many other problems and as a result, was withdrawn. The author has been working on fixing some of the bugs and it should be fixed in GNOME 2.0.1. This is bugzilla number bug 78007.

Some features formerly provided by viewports no longer available in Sawfish

In order to simplify sawfish code and user interface, viewports have been removed from Sawfish2. The current functionality replaces nearly all viewport functionality, except two-dimensional workspace navigation. This is bug number 82337.

Note

The following has been reported as a workaround when added to .sawfishrc: 

			  ;; Get viewports back
			  (setq customize-command-classes '(default viewport))
			  
			  ;; (setq viewport-dimensions '(NUMBER_OF_COLS . NUMBER_OF_ROWS))
			  (setq viewport-dimensions '(6 . 1))  ;; example
