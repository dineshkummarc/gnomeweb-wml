Kända problem

All programvara innehåller, när det släpps, fel som utvecklarna känner till men utav olika anledningar har valt att inte åtgärda innan själva utgåvan sker. Fri programvara är inte annorlunda gentemot proprietär programvara på denna punkt, förutom att med fri programvara berättar vi för användarna om dessa problem.

Vi uppmuntrar också användarna att rapportera fel så att de kan åtgärdas. Det bästa sättet att rapportera fel som hittats i GNOME är att använda den enkla felrapporteringsguiden. Denna kommer att ta dig igenom de steg som är nödvändiga för att avge en högkvalitativ felrapport, och försäkra dig om att felrapporten märks korrekt. Om du är alldeles för avancerad för att tåla något med ordet "enkelt" i sig finns det även det traditionella felrapporteringsformuläret. Mer information om de fel som redan rapporterats finns i vår Bugzilla. Bland de mest signifikanta problemen med GNOME 2.10:

Lista med kända problem

  • Fokus för nya fönster i Mozilla/Firefox: När du startar Mozilla eller Firefox och redan har ett annat Mozilla- eller Firefox-fönster öppet, kan det hända att det nya fönstret visas utan fokus och bakom det fönster som har fokus. GNOME:s mozilla-baserade webbläsare Epiphany har inte detta problem, och Mozilla-utvecklarna arbetar på att snabbt åtgärda detta problem i framtida versioner av Mozilla och Firefox.

  • Tillfällig låsning vid uppstart av session: Program med trasig sessionshantering kan orsaka att inloggningen låses i ett par minuter då inloggningsskärmen visas. Standardsessionen innehåller inga sådana program, så detta kommer endast att bli ett problem om du sparar din session vid utloggningen. Om du stöter på detta problem är det bara att vänta på att inloggningen fortsätter, och sedan ta bort alla sådana program innan du sparar din session på nytt.
